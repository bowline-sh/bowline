# Security

Please report vulnerabilities privately before opening a public issue.

Use GitHub private vulnerability reporting when it is enabled for the public
repo. If that is unavailable, email `security@crowlabs.dev` with a concise
summary and a safe way to reproduce the issue.

Do not include raw `.env` values, Recovery Key words, device private keys,
workspace keys, account tokens, or private repository contents unless a
maintainer explicitly asks for a secure transfer path.

Security-sensitive areas include device trust, Recovery Keys, project env sync,
workspace encryption, signed object access, Git-state continuity, and agent
lease permissions.
