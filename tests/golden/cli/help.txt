bowline command shell

Discovery:
  bowline help [topic] [--json]
  bowline version [--json]
  bowline contract [topic] [--summary] [--json]

Workspace:
  bowline update [--check] [--version <version>] [--json]
  bowline login [--headless] [--no-poll] [--json]
  bowline logout [--json]
  bowline status [--root <path>] [--project <path>] [--watch] [--all] [--json]
  bowline setup [--root <path>] [--socket <path>] [--json] | bowline setup <path> [--yes] [--socket <path>] [--json]
  bowline events [--root <path>] [--project <path>] [--limit <n>] [--json|--human|--quiet]
  bowline tui [--root <path>] [--project <path>] [--socket <path>]
  bowline forget-local <project> [--yes] [--json] [--dry-run]
  bowline archive <project> [--restore] [--json] [--dry-run]
  bowline purge <project> [--grace <days>|--cancel] [--json] [--dry-run]

Trust:
  bowline device approve [--root <path>] (--request <id>|--code <short-code>) [--yes] [--json] [--dry-run]
  bowline device deny [--root <path>] (--request <id>|--code <short-code>) [--json] [--dry-run]
  bowline device revoke [--root <path>] --device <id> [--json] [--dry-run]
  bowline recover <status|create|verify <id>|rotate|revoke <id>|use <id>> [--json] [--dry-run]
  bowline device list [--root <path>] [--json|--human|--quiet]
  bowline device request [--root <path>] [--json]
  bowline device accept [--root <path>] --request <id> [--json]

Remote:
  bowline connect <host> [--root <path>] [--binary <path>] [--json] [--dry-run]

Work:
  bowline work create [project] <name> [--from <restore-point>] [--json] [--dry-run]
  bowline work list [--all] [--json|--human|--quiet]
  bowline work diff [target] [--path <glob>...] [--json]
  bowline work review <target> [--path <glob>...] [--json]
  bowline work accept <target> [--path <glob>...] [--json] [--dry-run]
  bowline work discard <target> [--json] [--dry-run]
  bowline work restore <target> [--json] [--dry-run]
  bowline work cleanup [--apply] [--json] [--dry-run]

Daemon:
  bowline daemon start [--json]
  bowline daemon stop [--json]
  bowline daemon status [--json]
  bowline daemon install [--json] [--dry-run]
  bowline daemon restart [--json] [--dry-run]
  bowline daemon uninstall [--json] [--dry-run]

Support:
  bowline diagnostics collect --root <path> [--json]
  bowline doctor [--engine manifest] [--json]

Command options follow the complete command path. Inspect `bowline help <command>` for the exact definition.
